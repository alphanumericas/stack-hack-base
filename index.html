Note generali

Concetti fondamentali

Prima di leggere il codice, è utile richiamare il significato dei registri coinvolti e il comportamento dello stack.

%RBP — Base Pointer

Punta alla base del frame corrente dello stack. Le variabili locali vengono allocate sotto l'indirizzo da lui puntato (indirizzi di memoria più bassi).

%RSP — Stack Pointer

Punta alla cima dello stack (indirizzo più basso attualmente occupato). Ogni push decrementa RSP di 8 byte; ogni pop lo incrementa.

%RIP — Instruction Pointer

Contiene l'indirizzo dell'istruzione in esecuzione. Non è accessibile direttamente con mov: si usa leaq (%%rip), %reg. Scrivere in RIP causa Segmentation Fault.

Stack — Direzione

Lo stack parte dagli indirizzi alti e cresce verso quelli bassi. RBP segna il confine del frame; RSP si sposta ad ogni push/pop.

Return Address

Al momento di una call, l'indirizzo dell'istruzione successiva viene messo in cima allo stack. Sostituendolo con &func prima del ret, si controlla dove andrà il flusso.

Segmentation Fault

Errore generato dall'OS quando un programma accede a memoria non permessa. Scrivere direttamente in RIP tramite mov ne è un esempio classico.

ℹ️ Inline Assembly — AT&T syntax
Nell'inline assembly GCC/Clang la sintassi è AT&T: il sorgente viene prima, la destinazione dopo (mov src, dst), al contrario dell'Intel. I registri hanno il prefisso %% e i segnaposto per le variabili C++ sono %0, %1, …
ℹ️ Etichette nel codice macchina
I label (func:, main:) non esistono nel binario finale: esistono solo nello pseudo-assembly generato dal compilatore per debug. A runtime si passa direttamente da indirizzo a indirizzo.
Sezione 1 di 3
01

Print di RBP, RIP e Segfault

Si legge il valore corrente di RBP con mov e si ottiene l'indirizzo puntato da RIP con leaq (%%rip). Dereferenziando il puntatore a RIP si legge il codice macchina del programma. Si mostra anche perché tentare di scrivere in RIP via mov provoca un Segmentation Fault.

g++ clang++
📌 Nota chiave
leaq (%%rip), %0 non scrive dentro RIP: recupera l'indirizzo a cui RIP punta e lo copia nella variabile C++. Il vero RIP resta immutato.
⚠️ Segfault — scrittura in RIP
movq %0, (%%rip) tenta di scrivere direttamente nel registro RIP: l'OS lo intercetta e termina il processo con SIGSEGV. RIP può essere alterato legalmente solo con call, jmp o ret.
01_rip_rbp_segfault.cpp C++ / x86-64 ASM 
#include <iostream>
#include <cstdint>

int main(void)
{
    long* RIP;
    long* RBP;

    asm volatile (
        "leaq (%%rip), %0\n\t"
        "movq %%rbp,   %1\n\t"
        : "=r"(RIP), "=r"(RBP)
    );

    std::cout << "CELLA DI MEMORIA PUNTATA DA RBP: " << RBP << "\n";
    std::cout << "CELLA DI MEMORIA PUNTATA DA RIP: " << RIP << "\n";
    std::cout << std::endl;
    std::cout << "VALORE DENTRO LA CELLA DI MEMORIA PUNTATA DA RBP: " << std::hex << *RBP << "\n";
    std::cout << "VALORE DENTRO LA CELLA DI MEMORIA PUNTATA DA RIP: " << std::hex << *(RIP) << "\n";
}

📌 Perché differiscono?
g++ genera il prologo/epilogo standard (push rbp + pop rbp) anche per funzioni vuote. clang++ elimina il frame pointer e inserisce solo ret, seguito da NOP padding per allineare la funzione successiva a 16 byte.

02

Accedere a variabili tramite RBP

Si legge il valore di RBP, poi si percorre lo stack a ritroso (decrementando un puntatore) finché non si incontra l'indirizzo di una variabile locale specifica. A quel punto la si sovrascrive direttamente, senza mai usare il suo nome.

g++ clang++
📌 Perché questo funziona
Le variabili locali vivono nello stack frame in posizioni contigue. Conoscendo l'indirizzo di RBP e quello della variabile target, è sufficiente decrementare un puntatore di 8 byte per volta finché i due indirizzi coincidono — e poi scrivere lì.
⚠️ RBP non viene modificato
Anche quando il puntatore RBP (copia C++ del registro) punta temporaneamente allo stesso indirizzo di var, il registro hardware %%rbp rimane invariato — come dimostrato rileggendolo a fine programma.
02_memory_hack.cpp C++ / x86-64 ASM 
#include <iostream>
#include <cstdint>

int main()
{
    long* RBP;
    asm volatile (
        "mov %%rbp, %0"
        : "=r"(RBP)
    );

    std::cout << "RBP: " << RBP << "\n";
    long var = 10;

    while (RBP != &var) {
        RBP -= 1;
        std::cout << "rbp - 1, rbp = " << RBP << "\n";
    }

    *RBP = 20;
    std::cout << var << "\n";

    asm volatile (
        "mov %%rbp, %0"
        : "=r"(RBP)
    );
    std::cout << "RBP: " << RBP << "\n";
}

Sezione 3 di 3
03

Loop tramite Return Address

Si crea un loop infinito senza usare call o istruzioni di salto esplicite: si intercetta il return address sullo stack e lo si sovrascrive con l'indirizzo della funzione stessa. Al ret, il programma torna all'inizio della funzione.

g++ (GNUC) clang++
📌 Meccanismo — come funziona il return address
Quando call func viene eseguita, la CPU fa automaticamente push RIP poi salta a func. Al ret, quella word viene rimessa in RIP via pop. Sostituendola con &func prima del ret, si riporta il flusso all'inizio della funzione.
⚠️ Differenza g++ / clang++ — frame pointer
g++ genera prologo/epilogo espliciti (push rbp / pop rbp), quindi il return address non è in cima all'RSP all'ingresso. clang++ con alcune ottimizzazioni omette il frame pointer, rendendo il return address immediatamente accessibile via pop. Per questo il codice usa #ifdef __clang__.
📌 Memoria e ricorsività
Ogni iterazione non libera il frame precedente dallo stack, quindi la memoria cresce — esattamente come nella ricorsione classica.
03_loop_assembly.cpp C++ / x86-64 ASM 
#include <iostream>
#include <cstdint>

void print(uint64_t* rbp)
{
    uint64_t rax, rbx, rcx;
    uint64_t* rip;

    asm(
        "movq %%rax, %0\n\t"
        "movq %%rbx, %1\n\t"
        "movq %%rcx, %2\n\t"
        "leaq (%%rip), %3\n\t"
        : "=r"(rax), "=r"(rbx), "=r"(rcx), "=r"(rip)
    );

    std::cout << "RBP: " << rbp << "\n";
    std::cout << "RAX: " << rax << "\n";
    std::cout << "RBX: " << rbx << "\n";
    std::cout << "RCX: " << rcx << "\n";
    std::cout << "RIP: " << rip << "\n";
}

void func()
{
#ifdef __clang__
    uint64_t* rsp_iniziale;
    void* RET_ADDR;

    asm volatile (
        "leaq (%%rsp), %0\n\t"
        "pop %1\n\t"
        : "=r"(rsp_iniziale), "=r"(RET_ADDR)
    );

    RET_ADDR = (void*)&func;

    asm volatile (
        "mov %1, %%rsp\n\t"
        "push %0\n\t"
        : "=r"(RET_ADDR)
        : "r"(rsp_iniziale)
    );

    std::cout << "ritornado ad indirizzo " << (void*)&func << " | RSP: " << (uint64_t*)rsp_iniziale << "\n";

#elif __GNUC__
    void* RBP;
    void* RET_ADDR = (void*)&func;
    void* RSP_IA;

    asm volatile (
        "mov %%rsp, %0\n\t"
        : "=m"(RSP_IA)
    );

    std::cout << "ritornare a " << RET_ADDR << " | RSP: " << (long*)RSP_IA << "\n";

    asm volatile (
        "pop  %0\n\t"
        "push %1\n\t"
        "mov %2, %%rsp\n\t"
        "ret    \n\t"
        : "=m"(RBP), "=m"(RET_ADDR)
        : "m"(RSP_IA)
    );
#endif
}

int main()
{
    std::cout << "prima di func\n";
    func();
    std::cout << "dopo func\n";
    return 0;
}
